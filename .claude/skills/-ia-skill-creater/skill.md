# 根据信息分析课程创建技能的工作流程与方法论

## 核心任务定义

**目标**：将信息分析课程中的理论模型、认知模型和实践方法，转化为可执行的 Claude Code Skills，实现知识的自动化应用。

**本质**：这是一个"知识工程"过程，将抽象的思维模型转化为具体的、可执行的技能模块。

---

## 一、核心工作流程

### 阶段 1：信息理解与结构化（Input → Structure）

#### 1.1 课程材料收集
- **操作**：批量收集课程原始材料（DOC、MD、视频转录稿等）
- **工具**：markitdown（格式转换）、文件批量处理脚本
- **输出**：标准化的 Markdown 文件集合

**示例对话**：
```
用户：用markitdown 这个python 包，批量把所有信息分析第三期
文件夹当中的所有doc文件转换成.md 文件，用脚本批量实现
```

#### 1.2 课程内容整合
- **操作**：阅读并整合所有课程材料，提取核心概念和模型
- **策略**：使用多代理并行处理（explore 代理）应对大规模内容
- **输出**：完整的课程总结文档

**示例对话**：
```
用户：访问.sources/信息分析文第三期文件夹，理解信息分析课程
总结当中的认知模型，然后用explore 代理阅读信息分析第三期的
内容之，补充并且完善所有信息分析课程的认知模型
```

**实际成果**：
- 原始：~710 行，6 个主要部分
- 整合后：~1,490 行，12 个主要部分
- 新增：50+ 认知模型，30+ 方法论框架

### 阶段 2：价值识别与筛选（Structure → Value）

#### 2.1 应用元反空模型识别核心价值
使用**元反空思维框架**作为筛选器：

**元（Meta）维度**：寻找源头与高阶模型
- 最小全局认识构建法
- 信息分布六大数学原理
- 本体论构建技能
- V型学习法

**反（Reverse）维度**：逆向思维与反常识
- 批量论文获取法（反常识地下载1000篇）
- 反信息分析法
- 多重竞争性假设分析法
- 交叉验证法

**空（Empty）维度**：跳出框架与留白
- 结构阅读法
- 4×5清单法
- 信息加工留白法

**实用执行维度**：可直接操作的方法
- 四步走信息分析流程
- 公司网络分析法
- 知识管理系统化法

#### 2.2 生成 42 个核心技能清单
- **分组原则**：按元反空维度分组，每组 9 个，其他 6 个
- **筛选标准**：
  - ✅ 自动化可行性（是否可系统化）
  - 🎯 应用价值（是否高频使用）
  - 📚 知识密度（是否包含核心模型）
  - 🔗 互操作性（是否可组合）

**示例对话**：
```
用户：根据元反空模型，找到最有价值的42个能被转换成
claude code skills 的源技能，分成四个组，每个组最多9个，
还有其他，标明出处和文件位置
```

**输出格式**：
```markdown
### 技能名称
**英文名**: English Name
**出处**: `文件路径`
**位置**: 第X行
**核心**: 核心概念
**自动化**: ✅/⚠️ 可自动化程度
**Claude Code应用**: 应用场景
```

### 阶段 3：技能设计与创建（Value → Skill）

#### 3.1 选择目标技能
从清单中按优先级选择：
- **高优先级**：基础框架、核心工作流
- **中优先级**：效率提升工具
- **低优先级**：深度分析能力

#### 3.2 搜索相关材料
- **操作**：在现有材料中搜索关于该技能的所有理解
- **示例**：搜索"元反空"相关内容，整合：
  - 阳志平老师的原始定义
  - 信息分析课程中的应用
  - 实际案例

**示例对话**：
```

请搜索当前文件夹当中所有对元反空的理解
```

#### 3.3 使用 tutorial-extractor 技能提取教程
- **工具**：tutorial-extractor 技能
- **目的**：从对话和材料中提取可复用的模式
- **输出**：结构化的技能文档

**示例对话**：
```
用户：用tutorial-extractor这个技能
```

#### 3.4 应用 skill-creator 最佳实践
- **工具**：skill-creator 技能
- **核心原则**：
  1. **简洁是关键**：只添加 Claude 尚未拥有的上下文
  2. **设置适当自由度**：根据任务脆弱性匹配具体性
  3. **渐进式披露**：三级加载系统

**示例对话**：
```

以及触发和合适的脚本和refernece 等等
```

## 二、关键方法论和原则

### 2.1 元反空筛选法

**核心思想**：使用元反空模型本身来筛选和整理技能

**三个维度**：
- **元（Meta）**：高阶模型、源头理论、本质方法
- **反（Reverse）**：反常识、逆向操作、批判性思维
- **空（Empty）**：跳出框架、跨领域、留白思考

**筛选矩阵**：
```
元维度技能（9个）   → 寻找源头与高阶模型
反维度技能（9个）   → 逆向思维与反常识
空维度技能（9个）   → 跳出框架与留白
实用执行技能（9个） → 可直接操作的方法
其他高价值技能（6个）→ 补充性能力
```

### 2.2 渐进式披露原则

**三级加载系统**：
1. **元数据层**（~100 字）：名称 + 描述，始终在上下文中
2. **指令层**（<5k 字）：SKILL.md 主体，技能触发时加载
3. **资源层**（无限）：scripts/references/assets，按需加载

**核心优势**：
- 最小化上下文膨胀
- 提高技能加载效率
- 保持 SKILL.md 简洁

### 2.3 自动化可行性评估

**三级评估标准**：
- ✅ **完全可自动化**（62%）：可系统化执行、可算法化
- ⚠️ **部分可自动化**（38%）：需要人工判断、可辅助
- ❌ **难以自动化**（0%）：需要创造性思维

**评估维度**：
1. **流程确定性**：是否有明确步骤？
2. **输入标准化**：是否可用结构化输入？
3. **输出可验证**：是否有明确的成功标准？
4. **工具支持度**：是否有现成的 API/工具？

### 2.4 42 原则

**核心思想**：限制数量，聚焦高价值

**分组逻辑**：
- 每组 9 个（认知限制：工作记忆 4±1，但留有选择空间）
- 4 个主要组（元、反、空、实用）
- 1 个补充组（其他 6 个）

**筛选逻辑**：
1. 必须有明确的出处和位置
2. 必须有清晰的核心概念
3. 必须评估自动化可行性
4. 必须明确 Claude Code 应用场景

### 2.5 最小全局认识法

**应用在技能创建**：
- 20% 的技能覆盖 80% 的使用场景
- 优先创建高优先级技能（基础框架、核心工作流）
- 每个技能聚焦一个核心能力

## 五、关键模式和反模式

### 5.1 成功模式

#### 模式 1：渐进式披露
```markdown
# SKILL.md (简洁主体)
## Quick Start
[核心操作]

## Advanced Features
- **功能 1**：See [ADVANCED.md](references/ADVANCED.md)
- **功能 2**：See [EXAMPLES.md](references/EXAMPLES.md)
```

#### 模式 2：领域组织
```
skill/
├── SKILL.md (导航和概述)
└── references/
    ├── domain1.md (领域 1 详细信息)
    ├── domain2.md (领域 2 详细信息)
    └── domain3.md (领域 3 详细信息)
```

#### 模式 3：条件细节
```markdown
## Basic Usage
[基础内容]

## Advanced
**For [高级功能]**：See [ADVANCED.md](references/ADVANCED.md)
**For [特殊情况]**：See [SPECIAL.md](references/SPECIAL.md)
```

### 5.2 反模式

#### 反模式 1：过度解释
❌ **错误**：
```markdown
## 什么是 Python？
Python 是一种高级编程语言，由 Guido van Rossum 于 1991 年创建...
```

✅ **正确**：
```markdown
## Python Integration
使用 python-tool 执行 Python 脚本。
```

**原则**：假设 Claude 已经聪明，只提供它不知道的信息。

#### 反模式 2：重复信息
❌ **错误**：
```markdown
# SKILL.md
[详细的 API 文档]

# references/api.md
[同样的 API 文档]
```

✅ **正确**：
```markdown
# SKILL.md
## Quick Start
[核心用法]
详见 [API 文档](references/api.md)

# references/api.md
[完整的 API 文档]
```

**原则**：信息只存在于一个地方，另一个地方引用。

#### 反模式 3：第二人称描述
❌ **错误**：
```yaml
description: I can help you analyze data using advanced techniques...
```

✅ **正确**：
```yaml
description: Applies advanced data analysis techniques to extract insights.
Use when user needs data analysis, pattern recognition, or statistical modeling.
```

**原则**：始终使用第三人称。

## 六、核心方法论总结

### 6.1 元反空方法论

**在技能创建中的应用**：

**元（Meta）**：技能的本质是什么？
- 技能是"知识的程序化封装"
- 这个技能是否可以使用到更通用的场景
- 技能是"可复用的专家级工作流程"
- 技能是"Claude 的能力扩展包"

**反（Reverse）**：什么不是技能？
- 技能不是文档（FAQ、README）
- 技能不是教程（一次性学习材料）
- 技能不是完整解决方案（需要用户输入）
- 如果没有这个技能会怎样
- 如果这个技能最大的漏洞事什么，如何修补

**空（Empty）**：跳出技能看技能？
- 技能组合：多个技能协同工作
- 技能进化：根据使用反馈迭代
- 技能生态：构建技能之间的网络
- 如果没有这个技能，会怎样，是否有更优雅的方式来实现这个流程

### 6.2 最小全局认识法

**在技能创建中的应用**：

**时间维度**：
- 快速验证（创建 MVP 技能）
- 迭代优化（根据使用反馈）
- 长期演进（技能版本的更新）

**空间维度**：
- 覆盖核心场景（20% 技能覆盖 80% 用例）
- 预留扩展空间（技能组合能力）
- 生态完整性（互补技能）

**变量关系维度**：
- 技能粒度（太大难以复用，太小价值有限）
- 技能依赖（独立 vs 组合）
- 技能触发（自动 vs 手动）

### 6.3 交叉验证法

**在技能创建中的应用**：

**多源验证**：
- 课程材料验证（确认理解正确）
- 实际使用验证（测试技能效果）
- 用户反馈验证（收集改进建议）

**方法交叉**：
- 多个技能解决同一问题（提供选择）
- 不同工具完成同一任务（比较优劣）
- 多种表达方式（适应不同用户）

### 6.4 4×5 清单法

**在技能创建中的应用**：

**4 个主要组块**：
1. **信息收集与结构化**（准备阶段）
2. **价值识别与筛选**（设计阶段）
3. **技能实现与验证**（开发阶段）
4. **迭代与优化**（维护阶段）

**每个组块的 5 个关键步骤**：
1. **准备阶段**：收集 → 整理 → 标注 → 索引 → 标准化
2. **设计阶段**：理解 → 分析 → 筛选 → 排序 → 规划
3. **开发阶段**：设计 → 编写 → 测试 → 验证 → 打包
4. **维护阶段**：使用 → 反馈 → 分析 → 改进 → 发布

## 七、实践建议和最佳实践

### 7.1 技能创建原则

**1. 简洁是最高美德**
- 假设 Claude 已经聪明
- 只添加它不知道的信息
- 每一条信息都要质疑：值得 token 成本吗？

**2. 设置适当自由度**
- 高自由度：基于文本的指令（多种方法有效）
- 中自由度：带参数的伪代码（有首选模式）
- 低自由度：特定脚本（一致性至关重要）

**3. 渐进式披露设计**
- 元数据：始终加载（~100 字）
- SKILL.md：触发时加载（<5k 字）
- 资源：按需加载（无限）

**4. 可验证性优先**
- 明确的输入输出
- 清晰的成功标准
- 可测试的步骤

### 7.2 优先级策略

**高优先级技能（立即实现）**：
- 基础框架技能（如：最小全局认识构建法）
- 核心工作流（如：四步走信息分析流程）
- 质量控制（如：交叉验证法）

**中优先级技能（第二阶段）**：
- 效率提升工具（如：批量论文获取法）
- 数据分析能力（如：信息分布六大原理）
- 思维训练（如：多重竞争性假设分析法）

**低优先级技能（高级应用）**：
- 安全防御（如：反信息分析法）
- 创新思维（如：时空变形法）
- 个人发展（如：可能自我探索法）

### 7.3 质量保证清单

**完整性**：
- [ ] YAML frontmatter 包含必需字段

### 7.4 迭代优化策略

**快速迭代**：
