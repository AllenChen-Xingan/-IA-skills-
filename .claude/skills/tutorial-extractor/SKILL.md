# 教程提取器

把对话变成教程。一个教程，一个技巧，四个章节讲透。

**核心原则：** 四个章节讲不清？拆分它。

## When to Use

- 从 AI 聊天记录中提取技巧
- 记录问题解决过程
- 捕捉"原来如此"的顿悟瞬间

**不适用：** 基础 FAQ、项目配置、一次性修复

## Quick Reference

| 步骤 | 操作 |
|------|------|
| 发现 | 扫描对话找亮点 |
| 评估 | 可复用？够通用？ |
| 提取 | 套用 4 章节结构 |
| 打磨 | 通用化 + 加例子 |

## 四章节结构

```markdown
# [技巧名称]

## 1. 问题场景
- 什么情况触发这个技巧？
- 为什么现有方案不行

## 2. 核心技巧
1. 第一步
2. 第二步
3. 第三步

## 3. 实战案例
- 改造前：[情况]
- 改造后：[效果]

## 4. 避坑指南
- 错误做法：[常见错误]
- 正确做法：[应该怎么做]

---
版本：v1.0 | 来源：[出处]
```

## 淘金信号

在对话中寻找：
- "这招妙啊！" → 灵感迸发
- "还能这样？" → 发现隐藏功能
- "我之前做错了" → 纠错之金
- 多轮来回解决问题 → 复杂技巧

**快速测试：** 你会收藏这段对话吗？会 = 值得提取。

## 提取模式

**纠错之金：** 用户纠正 AI → 提取为"避坑指南"
**发现之金：** 发现现成工具 → 提取为"核心技巧"
**长征之金：** 多轮调试 → 提取为"症状→假设→洞察→方案"

## 通用化处理

```markdown
# 优化前（太具体）
"运行 `cd ~/my-project && npm run migrate`"

# 优化后（通用）
"运行迁移命令：`cd <项目根目录> && <迁移命令>`"
```

**脱敏清单：**
- API 密钥 → `sk-xxx-your-key`
- 真实姓名 → 通用角色
- 内部路径 → `<占位符>`

## Common Mistakes

| 错误 | 修复 |
|------|------|
| 塞太多技巧 | 拆成多个教程 |
| 没真实例子 | 加前后对比 |
| 跳过"为什么" | 解释道理 |
| 忘了脱敏 | 跑脱敏清单 |

## 质量检验

**5 秒测试：** 同事能从标题 + 第一段在 5 秒内理解技巧吗？

**复制粘贴测试：** 例子代码能直接复制、替换占位符、立即运行吗？

**"那又怎样"测试：** 每章节写完问自己"那又怎样？"说不清价值就重写。
